V2: Native OSD & RTSP

Phase 6 introduces a fully hardware-accelerated pipeline. It draws target boxes via nvdsosd in GPU memory and streams video via RTSP. It optionally includes a DeepStream object detector (PGIE) to let the user securely click on the object they want to track in a window showing the detector's suggestions.

What is PGIE?

In DeepStream, a Primary GIE (PGIE) is a nvinfer element that runs an object detector inline in the GStreamer pipeline. It attaches detection results directly to the frame buffer as NvDsObjectMeta records, which any downstream element or pad probe can read.

Phase 6 uses the pre-built ResNet-10 INT8 detector that ships with DeepStream SDK. It detects four classes (Car, Bicycle, Person, Roadsign) and produces NvDsObjectMeta with rect_params (pixel bounding box), class_id, obj_label, and confidence.

Using a Custom Detector
The PGIE element is fully configurable. To use a different detector (e.g., PeopleNet, YOLOv8, or a custom model), replace the paths in pgie_config.txt: model-file, model-engine-file, labelfile-path, and update num-detected-classes and output-blob-names. No Python code changes are required.

Phase 6 Pipeline — NvDsOsd + RTSP + PGIE

The recommended V2 application is deepstream_rtsp_app.py. It uses a native DeepStream pad probe instead of appsink, draws boxes via nvdsosd in GPU memory, and streams output over RTSP using hardware H.264 encoding.

Pad Probe Logic — tracker_probe()

The pad probe is attached to the sink pad of nvdsosd. It fires after nvinfer has already populated frame metadata, but before nvdsosd renders the frame. This ordering ensures PGIE detections are available when the probe reads them on frame 0.

1

First Frame — Capture & Signal

On frame 0, the probe calls pyds.get_nvds_buf_surface() to get an RGBA NumPy array, reads all NvDsObjectMeta entries from obj_meta_list (populated by nvinfer), stores them in state.first_frame_detections, and calls state.frame_ready.set() to wake the main thread. Then it blocks on state.init_done.wait() until ROI selection is complete.

2

Main Thread — ROI Selection (while probe waits)

The main thread waits on state.frame_ready.wait(), then calls select_bbox_click_to_select() or manual_roi_select(). All OpenCV GUI calls happen here — never inside the probe (Lesson 25). Once a box is selected, state.init_done.set() unblocks the probe.

3

Subsequent Frames — Track & Draw

The probe converts the RGBA frame to RGB, calls TrackerManager.update(), and for each returned bounding box acquires an NvDsObjectMeta from the pool, fills rect_params and text_params, and adds it to the frame. nvdsosd then renders these boxes in GPU memory before the tee.

RTSP Streaming

After nvdsosd, a tee element splits the stream into two branches. The RTSP branch encodes with nvv4l2h264enc (Jetson NVENC hardware block) and sends RTP packets to a udpsink on port 5400. A GstRtspServer instance wraps a udpsrc media factory and serves the combined stream at rtsp://<jetson-ip>:8554/sutrack.

The display branch tries nv3dsink (Jetson-native, no EGL copy) first, falling back to nveglglessink with a nvvideoconvert(compute-hw=1) bridge to avoid the NVMM surface array crash (Lesson 24).

Headless / SSH Mode
Set pipeline.headless: true in tracker_config.yml. The display branch connects to fakesink instead of nv3dsink. No DISPLAY environment variable is needed. All output goes to the RTSP stream. Combine with tracker.static_roi for fully unattended operation.

One-Shot GPU Optimization (Power-Save)

Running an object detector on every frame is expensive. To maximize performance, Phase 6.5 implements a "One-Shot" detection flow:

  • Frame 0: Detector runs with interval=0 to provide ROI candidates.
  • Selection: User clicks an object, the tracker initializes.
  • Optimization: If state.pgie_one_shot is true (default), the application dynamically sets the PGIE interval property to 10,000 frames.

Use --no-one-shot to keep the detector running in the background.

This effectively puts the detector to sleep immediately after it completes its task, freeing 100% of the GPU inference cycles for the SUTrack model.

Metadata Filtering (Clean Output)

Even if the detector is "sleeping", its initial detections on Frame 0 might persist, or it might fire occasionally. To ensure a professional, distraction-free output, the tracker_probe aggressively strips all detector metadata once tracking starts.

Lesson 27 — Explicit Metadata Removal
Simply ignoring detector boxes in Python isn't enough. The nvdsosd element draws every box it finds in obj_meta_list. The probe must explicitly call pyds.nvds_remove_obj_meta_from_frame() for every frame to keep the stream clean.

Thread Synchronization — Why It Matters

GStreamer runs the pad probe in a streaming thread. OpenCV GUI functions (namedWindow, imshow, waitKey, setMouseCallback) must only be called from the Python main thread. Calling them from the probe causes a segfault or a silent hang on Jetson.

Phase 6 uses two threading.Event objects to bridge the threads safely:

THREADING.EVENT SYNCHRONIZATION (Lessons 25, 26) GStreamer streaming thread Python main thread (tracker_probe callback) (main() function) ───────────────────────── ────────────────────────────── pipeline starts playing --> loop.run() is blocked normally frame 0 arrives: read RGBA frame read PGIE detections state.frame_ready.set() --> main thread unblocks state.init_done.wait() select_bbox_click_to_select() [streaming BLOCKED] or manual_roi_select() manager.initialize(frame_rgb, bbox) <-- state.init_done.set() probe unblocks return Gst.PadProbeReturn.OK ───────────────────────── frame 1, 2, 3 ... → normal tracking loop
Lesson 25 — OpenCV GUI in Main Thread Only
All cv2 window/display calls must be in main(), never in tracker_probe(). The probe runs in GStreamer's internal streaming thread. Qt requires GUI calls on the OS main thread.
Lesson 26 — threading.Event for Probe / GUI Sync
Use state.frame_ready.set() in the probe to signal the main thread, and state.init_done.wait() to block the probe until the user completes ROI selection. This keeps the streaming thread paused at exactly frame 0 with no busy-wait.

Click-to-Select Logic — select_bbox_click_to_select()

This function is called by main() after the probe signals frame 0 is ready and PGIE has provided detections.

1

Draw Detection Boxes

The first frame (RGBA) is converted to BGR. Each detection from state.first_frame_detections is drawn as a coloured rectangle with the class label and confidence score. A text prompt — "CLICK object to track. Q/ESC = manual ROI" — is overlaid at the top.

2

Open Window and Register Mouse Callback

cv2.namedWindow() + cv2.imshow() open the window. cv2.setMouseCallback() registers on_click(), which checks all detections when a left button press occurs.

3

Click Point to Bounding Box

For a click at pixel (px, py), the code checks every detection box (dx, dy, dw, dh). A box contains the click if dx <= px <= dx+dw and dy <= py <= dy+dh. If multiple boxes overlap, the one with the smallest area (dw * dh) wins — this selects the innermost box and avoids accidentally targeting a large background detection.

4

Return Selected Box or None

If a valid box is clicked, the window closes and returns [x, y, w, h]. Pressing Q or ESC returns None, which triggers the manual ROI fallback. The window stays open if the click was outside all detection boxes — the user can try again. 

def select_bbox_click_to_select(frame_rgba, detections):
    frame_bgr = cv2.cvtColor(frame_rgba, cv2.COLOR_RGBA2BGR)

    # Draw per-class coloured boxes + confidence labels
    for det in detections:
        x, y, w, h = det['x'], det['y'], det['w'], det['h']
        color = _CLASS_COLORS[det['class_id'] % len(_CLASS_COLORS)]
        cv2.rectangle(frame_bgr, (x, y), (x+w, y+h), color, 2)
        cv2.putText(frame_bgr, '%s %.2f' % (det['label'], det['confidence']),
                    (x, max(y-4, 12)), cv2.FONT_HERSHEY_SIMPLEX, 0.55, color, 1)

    cv2.putText(frame_bgr, 'CLICK object to track.  Q/ESC = manual ROI.',
                (10, 30), cv2.FONT_HERSHEY_SIMPLEX, 0.75, (0, 220, 255), 2)

    win_name = 'Click-to-Select Object'
    cv2.namedWindow(win_name, cv2.WINDOW_AUTOSIZE)
    cv2.imshow(win_name, frame_bgr)

    result = {'selected': None, 'done': False}

    def on_click(event, cx, cy, flags, param):
        if event != cv2.EVENT_LBUTTONDOWN:
            return
        best, best_area = None, float('inf')
        for det in detections:
            dx, dy, dw, dh = det['x'], det['y'], det['w'], det['h']
            if dx <= cx <= dx+dw and dy <= cy <= dy+dh:
                area = dw * dh
                if area < best_area:
                    best_area, best = area, [dx, dy, dw, dh]
        if best is not None:
            param['selected'] = best
            param['done']     = True

    cv2.setMouseCallback(win_name, on_click, result)

    while not result['done']:
        key = cv2.waitKey(50) & 0xFF
        if key == ord('q') or key == 27:   # Q or ESC
            break

    cv2.destroyWindow(win_name)
    cv2.waitKey(1)
    return result['selected']   # None if cancelled

Three-Tier Fallback Chain

The ROI selection logic in main() always falls back gracefully, so the application never crashes regardless of the environment.

ROI SELECTION FALLBACK CHAIN Is static_roi set in config (or --init_bbox on CLI)? YES --> Use it directly. No window opened. (headless / SSH safe) | NO | Is PGIE enabled AND nvinfer produced detections on frame 0? YES --> Open "Click-to-Select Object" window. | User clicks a detection box. | Box returned --> initialize tracker. | User presses Q/ESC --> falls through to manual. | NO (--no-pgie, or detections=0, or user pressed ESC) | v Open manual selectROI window. User draws a rectangle, presses SPACE/ENTER. Box returned --> initialize tracker. User presses ESC --> app exits gracefully.
1

Static ROI — Fully Headless

Set tracker.static_roi: "X,Y,W,H" in YAML or pass --init_bbox X Y W H. No display needed. Ideal for automated pipelines and SSH.

2

PGIE Click-to-Select — One Click

Requires a display (export DISPLAY=:0). The detector provides perfectly aligned boxes — no need to draw freehand. One click starts tracking.

3

Manual selectROI — Draw a Box

Fallback for any situation where PGIE is off or no detections appear. Classic mouse-drag interface. imshow() is called before selectROI() to initialize the Qt window handler (Lesson 9).

This is a standard DeepStream nvinfer config file. The pre-built ResNet-10 INT8 engine ships with JetPack at the path shown below.

num-detected-classes=4
interval=0                               # run every frame
gie-unique-id=1

KeyMeaning
model-engine-filePre-compiled TRT INT8 engine for ResNet-10 (batch=1). Ships with DeepStream SDK — no compilation needed.
network-mode=1INT8 quantized inference — fast on Jetson DLA/GPU without GPU memory penalty.
interval=0Run detection every frame. Can be increased (e.g., interval=5) once tracking is active to save GPU cycles.
cluster-mode=2DBSCAN clustering for bounding box post-processing (matches DS sample defaults).
pre-cluster-thresholdMinimum confidence before NMS. Lowering detects more objects but also more false positives.
output-blob-namesSpecific to ResNet-10 Caffe architecture. Must match if you replace the model.

CLI Usage

FlagTypeDescription
--pgie-config PATH str Override the pgie_config path from YAML. Use an absolute path or a path relative to the working directory.
--no-pgie flag Disable PGIE for this run regardless of tracker_config.yml. The pipeline skips nvinfer and goes straight to manual ROI selection. 
# Standard run — PGIE click-to-select (config enables PGIE)
export DISPLAY=:0
python deepstream/apps/deepstream_rtsp_app.py \
    --config deepstream/configs/tracker_config.yml \
    --input /path/to/video.mp4

# Disable PGIE at runtime, fall through to manual draw-a-box
python deepstream/apps/deepstream_rtsp_app.py \
    --config deepstream/configs/tracker_config.yml \
    --input /path/to/video.mp4 \
    --no-pgie

# Custom PGIE config (e.g., PeopleNet model)
python deepstream/apps/deepstream_rtsp_app.py \
    --config deepstream/configs/tracker_config.yml \
    --input /path/to/video.mp4 \
    --pgie-config /opt/nvidia/deepstream/deepstream/samples/configs/tlt_pretrained_models/config_infer_primary_peoplenet.txt
Headless / SSH with static_roi
When running over SSH without a display, always provide a static ROI so the app never attempts to open a GUI window: 
# In tracker_config.yml:
tracker:
  static_roi: "300,200,120,180"   # X,Y,W,H
With static_roi set, neither PGIE click-to-select nor manual draw-a-box is triggered — the pipeline initializes automatically on the first frame.
