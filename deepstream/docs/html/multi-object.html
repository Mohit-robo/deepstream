Multi-Object Manager

How multiple objects are tracked simultaneously, how detections are matched to existing tracks, and how lost tracks are cleaned up.

Overview

TrackerManager sits between the GStreamer pipeline and the individual TrackerInstance objects. Its job is to:

  • Keep a pool of active TrackerInstance objects, one per tracked object
  • Call each tracker's track() method every frame (sequential, shared engine)
  • Match incoming detector bounding boxes to existing tracks using IoU
  • Spawn new trackers for unmatched detections
  • Remove trackers that haven't been matched for too long
TRACKERMANAGER POSITION IN THE STACK AppState (callback) | v TrackerManager.initialize() called once when user picks ROI TrackerManager.update() called every frame | +--- TrackerInstance[0].track(frame, engine) +--- TrackerInstance[1].track(frame, engine) (sequential, shared engine) +--- TrackerInstance[N].track(frame, engine) | v IoU match new detections to existing tracks | +--- matched: update last_seen_frame (keep tracker alive) +--- unmatched + high confidence: spawn new TrackerInstance | remove_stale() delete trackers not seen for > max_age frames | v return { obj_id: [x, y, w, h], ... }

Tracker Lifecycle

1

Born

A new TrackerInstance is created via initialize() — either from manual user selection, or when a detector reports a new high-confidence object that doesn't match any existing track.

2

Active — tracked every frame

update() calls track() on every active instance each frame. The instance updates its state and last_seen_frame.

3

Matched by detector (optional, Phase 4.3)

When a PGIE detector is attached, its bounding boxes are matched to existing tracks by IoU. A successful match refreshes last_seen_frame, keeping the tracker alive even if the pure-tracking confidence drops.

4

Stale — too many frames without a match

If a tracker's last_seen_frame falls more than max_age frames behind the current frame, it is deleted. Default: 30 frames.

initialize() — Creating a New Tracker

def initialize(self, frame_rgb, bbox, frame_idx=0) -> int:
    obj_id = self._next_id
    self._next_id += 1

    inst = TrackerInstance(
        object_id      = obj_id,
        template_factor= self.template_factor,
        search_factor  = self.search_factor,
        template_size  = self.engine.template_size,
        search_size    = self.engine.search_size,
        feat_sz        = self.engine.feat_sz,
        use_hanning    = self.use_hanning,
    )
    inst.initialize(frame_rgb, bbox, frame_idx)
    self.active_trackers[obj_id] = inst
    return obj_id

Object IDs are assigned from a monotonically increasing counter _next_id. IDs are never reused — if tracker ID 2 is removed and a new object appears, it gets ID 3. This prevents ID confusion in downstream consumers.

update() — The Per-Frame Loop

def update(self, frame_rgb, frame_idx,
           detections=None) -> dict:

    results = {}

    # Run tracking for every active tracker (sequential — shared engine)
    for obj_id, inst in list(self.active_trackers.items()):
        try:
            bbox = inst.track(frame_rgb, self.engine, frame_idx)
            results[obj_id] = bbox
        except Exception as e:
            # Remove faulty tracker rather than crashing the whole pipeline
            del self.active_trackers[obj_id]

    # Optional: match detector boxes and spawn new trackers
    if detections:
        self._match_and_spawn(frame_rgb, frame_idx, detections, results)

    self.remove_stale(frame_idx)
    return results
Sequential Inference — Why Not Parallel?
All TrackerInstance objects share one SUTrackEngine (one TRT context, one set of GPU buffers). Parallelism would require multiple engines and multiply GPU memory usage by the number of tracked objects. Sequential is correct and efficient for typical tracker counts (1–5 objects).

IoU Matching — Connecting Detections to Tracks

When a detector (Phase 4.3 PGIE) provides bounding boxes for the current frame, the manager must decide: does each detection correspond to an object we are already tracking, or is it a new object?

The answer is computed with Intersection over Union (IoU): how much do the detector's box and the tracker's predicted box overlap?

IoU MATCHING ALGORITHM Active tracks: T0:[100,50,60,80] T1:[300,200,50,70] New detections: D0:[105,52,58,79] D1:[400,100,40,60] D2:[295,198,55,75] Compute IoU matrix: D0 D1 D2 T0: 0.92 0.01 0.04 T1: 0.00 0.03 0.88 Greedy match (highest IoU first, threshold=0.3): T0 -> D0 (IoU=0.92) matched D0 marked as used T1 -> D2 (IoU=0.88) matched D2 marked as used Unmatched detection: D1 (IoU with all tracks below 0.3) confidence(D1) >= min_confidence ? YES --> spawn new TrackerInstance for D1 NO --> ignore D1
# Greedy IoU matching inside update()
matched_det_indices = set()

for obj_id in tracked_ids:
    best_iou, best_det_idx = 0.0, -1
    for i, det in enumerate(detections):
        if i in matched_det_indices:
            continue
        iou = compute_iou(results[obj_id], det['bbox'])
        if iou > best_iou:
            best_iou, best_det_idx = iou, i

    if best_iou >= self.iou_match_threshold:
        matched_det_indices.add(best_det_idx)
        self.active_trackers[obj_id].last_seen_frame = frame_idx   # keep alive

# Spawn new trackers for unmatched high-confidence detections
for i, det in enumerate(detections):
    if i not in matched_det_indices:
        if det.get('confidence', 1.0) >= self.min_confidence:
            self.initialize(frame_rgb, det['bbox'], frame_idx)
ParameterDefaultMeaning
iou_match_threshold0.3Minimum IoU to consider a detection matched to an existing track. Lower = more permissive matching.
min_confidence0.25Minimum detector confidence to spawn a new tracker. Prevents spawning trackers on weak detections.
max_age30Frames a tracker survives without being matched before deletion.

Stale Tracker Removal — remove_stale()

When an object leaves the frame or becomes occluded, the tracker keeps running — it may drift to a wrong position. After max_age frames without a detector match, the tracker is removed. 

def remove_stale(self, current_frame: int):
    stale = [oid for oid, inst in self.active_trackers.items()
             if (current_frame - inst.last_seen_frame) > self.max_age]
    for oid in stale:
        del self.active_trackers[oid]

In Phase 4.2 (manual init, no detector), last_seen_frame is updated every time track() succeeds — so trackers are only removed if track() raises an exception. In Phase 4.3 (with PGIE detector), last_seen_frame is refreshed on IoU matches, allowing the detector to "rescue" a tracker that is drifting.

IoU Calculation — compute_iou()

IoU (Intersection over Union) measures how much two boxes overlap. A value of 1.0 means perfect overlap; 0.0 means no overlap at all.

IoU FORMULA Box A: [ax1, ay1, aw, ah] Box B: [bx1, by1, bw, bh] Intersection: ix1 = max(ax1, bx1) iy1 = max(ay1, by1) ix2 = min(ax1+aw, bx1+bw) iy2 = min(ay1+ah, by1+bh) inter_area = max(0, ix2-ix1) * max(0, iy2-iy1) Union: union_area = area(A) + area(B) - inter_area IoU = inter_area / union_area Example: A = [100, 50, 60, 80] area = 4800 B = [110, 55, 60, 80] area = 4800 Intersection = 50 * 70 = 3500 Union = 4800 + 4800 - 3500 = 6100 IoU = 3500 / 6100 = 0.574
def compute_iou(box_a: list, box_b: list) -> float:
    ax1, ay1, aw, ah = box_a
    bx1, by1, bw, bh = box_b

    inter_x1 = max(ax1, bx1)
    inter_y1 = max(ay1, by1)
    inter_x2 = min(ax1 + aw, bx1 + bw)
    inter_y2 = min(ay1 + ah, by1 + bh)

    inter = max(0.0, inter_x2 - inter_x1) * max(0.0, inter_y2 - inter_y1)
    union = aw * ah + bw * bh - inter
    return inter / union if union > 0 else 0.0

num_active Property

A simple read-only property used by the display code to show how many objects are currently being tracked in the FPS counter. 

@property
def num_active(self) -> int:
    return len(self.active_trackers)
