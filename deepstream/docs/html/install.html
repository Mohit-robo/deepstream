Installation Guide

Complete step-by-step installation from a bare JetPack image to a running DeepStream SUTrack app. No PyTorch required at runtime.

System Requirements

ComponentRequirementHow Installed
Hardware NVIDIA Jetson (Orin / Xavier / AGX Xavier) —
JetPack 5.x (L4T r35 or r36) SDK Manager / SDKM flash
DeepStream SDK 6.x or 7.x (must match JetPack) apt-get install deepstream-7.0
Python 3.8 – 3.10 Pre-installed with JetPack
TensorRT >= 8.5 Pre-installed with JetPack
CUDA >= 11.4 Pre-installed with JetPack
pyds Matching DeepStream version pip install pyds-*.whl — Step 3
GStreamer + RTSP >= 1.14 apt-get — Step 2
cuda-python >= 11.7 pip install cuda-python — Step 4

Step 1 — Verify Your Base System

Run these checks before installing anything to confirm JetPack, DeepStream, and TensorRT are already present on your Jetson. 

# Show JetPack / L4T version (should print something like "# R35 (release), REVISION: 4.1")
cat /etc/nv_tegra_release

# Confirm DeepStream is installed and operational
deepstream-app --version-all

# Confirm Python version (must be 3.8 – 3.10)
python3 --version

# Confirm TensorRT (bundled with JetPack — no separate install needed)
python3 -c "import tensorrt as trt; print('TRT', trt.__version__)"

# Confirm CUDA (bundled with JetPack)
nvcc --version
DeepStream Not Found?
If deepstream-app is missing, install it: 
sudo apt-get install -y deepstream-7.0
# or deepstream-6.4 depending on your JetPack version
Then re-run deepstream-app --version-all to confirm.

Step 2 — System Packages (GStreamer + RTSP Server)

Install GStreamer Python bindings, plugin packs, and the GstRtspServer library. The RTSP server library (gir1.2-gst-rtsp-server-1.0) is required for Phase 5+ RTSP streaming and is easy to forget. 

sudo apt-get update
sudo apt-get install -y \
    python3-gi \
    python3-gst-1.0 \
    gstreamer1.0-tools \
    gstreamer1.0-plugins-base \
    gstreamer1.0-plugins-good \
    gstreamer1.0-plugins-bad \
    gstreamer1.0-rtsp \
    libgstrtspserver-1.0-dev \
    gir1.2-gst-rtsp-server-1.0
PackageWhy It Is Needed
python3-giGObject Introspection — lets Python call GStreamer C libraries
python3-gst-1.0GStreamer Python overrides (Gst.init, Gst.Pipeline, etc.)
gstreamer1.0-toolsgst-launch-1.0, gst-inspect-1.0 for debugging
gstreamer1.0-plugins-base/good/badCore demuxers, decoders, and muxers
gstreamer1.0-rtspRTSP client plugin (rtspsrc) for RTSP input sources
libgstrtspserver-1.0-devC headers — required to build the GObject Introspection data
gir1.2-gst-rtsp-server-1.0Python bindings for GstRtspServer — required to create an RTSP server in Python

Verify:

python3 -c "
import gi
gi.require_version('Gst', '1.0')
gi.require_version('GstRtspServer', '1.0')
from gi.repository import Gst, GstRtspServer
Gst.init(None)
print('OK GStreamer', Gst.version_string())
print('OK GstRtspServer')
"

Step 3 — DeepStream Python Bindings (pyds)

pyds is the Python API for reading and writing DeepStream metadata (NvDsFrameMeta, NvDsObjectMeta, etc.). It ships as a pre-built wheel inside the DeepStream SDK installation. 

# Locate the wheel (path includes DS version number)
find /opt/nvidia/deepstream -name "pyds*.whl" 2>/dev/null

# Install — the glob expands to the correct versioned filename
pip install /opt/nvidia/deepstream/deepstream/lib/pyds-*.whl
Wheel Not Found?
If the wheel is missing, your DeepStream SDK installation may be incomplete. Common fix: 
sudo apt-get install --reinstall deepstream-7.0
Then search again with find /opt/nvidia/deepstream -name "pyds*.whl".

Verify:

python3 -c "import pyds; print('OK pyds')"

Step 4 — Python Runtime Packages

These are the only Python packages that need to be installed via pip. PyTorch is not required at runtime — all tracker preprocessing uses pure NumPy. 

pip install numpy opencv-python pyyaml cuda-python
PackageTested VersionWhy It Is Needed
numpy >= 1.21 All tracker preprocessing (crop, normalize, hann window, argmax) is pure NumPy
opencv-python >= 4.5 ROI selection window (selectROI), RGBA → BGR color conversion, PGIE click-to-select overlay
pyyaml >= 5.4 Load tracker_config.yml at startup
cuda-python >= 11.7 cuda.cudart API — allocates pinned host + device buffers and drives TRT inference. Replaces pycuda for better Jetson compatibility.
Why cuda-python instead of pycuda?
pycuda requires a compiler and often fails on Jetson due to architecture mismatches. cuda-python is a thin ctypes wrapper around the CUDA runtime library (libcudart.so) that is already installed with JetPack — no compiler needed.

Verify:

python3 -c "from cuda import cudart; print('OK cudart')"
python3 -c "import numpy; print('OK numpy', numpy.__version__)"
python3 -c "import cv2; print('OK OpenCV', cv2.__version__)"
python3 -c "import yaml; print('OK pyyaml')"

Step 5 — Full Verification

Run this script to check all 8 dependencies at once. Every line must print OK. Any FAIL must be fixed before running the app. 

python3 - <<'EOF'
checks = [
    ("pyds",           "import pyds"),
    ("tensorrt",       "import tensorrt as trt; print(trt.__version__, end='')"),
    ("cuda.cudart",    "from cuda import cudart"),
    ("numpy",          "import numpy"),
    ("opencv",         "import cv2"),
    ("pyyaml",         "import yaml"),
    ("GStreamer",      "import gi; gi.require_version('Gst','1.0'); from gi.repository import Gst; Gst.init(None)"),
    ("GstRtspServer",  "import gi; gi.require_version('GstRtspServer','1.0'); from gi.repository import GstRtspServer"),
]
for name, code in checks:
    try:
        exec(code)
        print(f"  OK  {name}")
    except Exception as e:
        print(f"  FAIL {name}: {e}")
EOF

Expected output:

  OK  pyds
  OK  tensorrt  8.x.x
  OK  cuda.cudart
  OK  numpy
  OK  opencv
  OK  pyyaml
  OK  GStreamer
  OK  GstRtspServer

Step 6 — Build the TensorRT Engine (one time only)

The app requires a compiled sutrack_fp32.engine file at the repo root. This step is only needed once per Jetson device — TRT engines are not portable across different GPU architectures.

Option A — Build from ONNX (requires full SUTrack repo)

# From the SUTrack repo root
cd ~/SUTrack

# Export the ONNX model (requires PyTorch + SUTrack dependencies)
python tracking/export_onnx.py --param sutrack_t224

# Compile the TRT engine (FP32 -- do NOT add --fp16)
bash deepstream/scripts/build_engine.sh sutrack.onnx sutrack_fp32.engine

Option B — Copy a Pre-Built Engine

If you already have a compiled engine from another session or machine with the same GPU architecture, copy it to the repo root: 

cp /path/to/sutrack_fp32.engine ~/SUTrack/sutrack_fp32.engine
Critical: FP32 Only — Never Use --fp16
The SUTrack decoder relies on sigmoid activations throughout. In FP16 precision, these underflow to near zero — the tracker sees no response peaks and outputs a broken, wandering bounding box. Always build without --fp16.

Verify the engine file exists and is non-zero:

ls -lh ~/SUTrack/sutrack_fp32.engine
# Expected: -rw-r--r-- 1 phoenix phoenix  ~52M  sutrack_fp32.engine

Optionally load it to confirm TRT can parse it:

python3 - <<'EOF'
import tensorrt as trt
logger = trt.Logger(trt.Logger.WARNING)
with open("sutrack_fp32.engine", "rb") as f:
    rt = trt.Runtime(logger)
    engine = rt.deserialize_cuda_engine(f.read())
print(f"OK engine — {engine.num_bindings} bindings")
EOF

Troubleshooting

SymptomLikely CauseFix
ModuleNotFoundError: pyds Wheel not installed pip install /opt/nvidia/deepstream/.../pyds-*.whl — see Step 3
ModuleNotFoundError: No module named 'gi' python3-gi not installed sudo apt-get install python3-gi python3-gst-1.0
ValueError: Namespace GstRtspServer not available GIR bindings for RTSP server missing sudo apt-get install gir1.2-gst-rtsp-server-1.0
ImportError: cuda-python or from cuda import cudart fails cuda-python not installed pip install cuda-python
deepstream-app: command not found DeepStream SDK not installed sudo apt-get install deepstream-7.0 (match your JetPack)
pyds wheel not found at /opt/nvidia/deepstream/... Incomplete DeepStream install sudo apt-get install --reinstall deepstream-7.0 then search again
import cv2 fails opencv-python not installed or wrong variant pip install opencv-python. On headless servers also try opencv-python-headless
TRT engine loads but gives wrong output Engine built with --fp16 Rebuild without --fp16 — see Step 6
Engine loads but binding count is wrong Old engine from different model variant Rebuild from sutrack_t224 ONNX. Expected: 3 inputs + 3 outputs = 6 bindings
RTSP stream never starts (rtsp_enabled: true) gir1.2-gst-rtsp-server-1.0 missing sudo apt-get install gir1.2-gst-rtsp-server-1.0 libgstrtspserver-1.0-dev
