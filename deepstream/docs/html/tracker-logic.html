Tracker Logic

The core algorithm that makes tracking work: how frames are preprocessed, how the model outputs are decoded into bounding boxes, and the critical rules that keep predictions stable.

How SUTrack Works — The Big Picture

SUTrack is a Siamese-style tracker. It takes two images on every frame:

  • Template — a small crop of the target from the initialization frame. This never changes.
  • Search region — a larger crop centered on the target's last known position. This updates every frame.

The model compares the template to every location in the search region and outputs a score map — a 14×14 grid of confidence values. The peak of that grid is where the target is most likely to be.

SUTRACK FORWARD PASS Template crop (112x112) Search crop (224x224) | | v v preprocess() preprocess() (1, 6, 112, 112) (1, 6, 224, 224) | | +----------+--------------+ | TRT engine | +----------+--------------+ | | | v v v score_map size_map offset_map (14x14) (2x196) (2x196) | hanning * spatial suppression (before argmax!) | argmax --> peak location (idx_x, idx_y) | decode cx, cy, w, h | map crop coords --> image coords | position-only clamp | updated [x, y, w, h]

Image Preprocessing — preprocess()

Both the template crop and the search crop are preprocessed the same way before being fed to the TRT engine. 

# ImageNet mean and standard deviation (used during training)
_MEAN = np.array([0.485, 0.456, 0.406], dtype=np.float32)
_STD  = np.array([0.229, 0.224, 0.225], dtype=np.float32)

def preprocess(img_rgb: np.ndarray) -> np.ndarray:
    img  = img_rgb.astype(np.float32) / 255.0   # [0,255] -> [0,1]
    img  = (img - _MEAN) / _STD                    # normalize per channel
    img  = img.transpose(2, 0, 1)                # HWC -> CHW  (3, H, W)
    img6 = np.concatenate([img, img], axis=0)    # CHW -> (6, H, W)
    return np.ascontiguousarray(img6[np.newaxis], dtype=np.float32)
    #  shape: (1, 6, H, W)

Three things happen here:

1

Normalize to [0, 1] then standardize

The model was trained with ImageNet normalization. Pixel values divided by 255 first, then shifted by the per-channel mean and divided by the standard deviation. This matches what the model saw during training.

2

Transpose HWC to CHW

NumPy images are (H, W, 3). Neural networks (and TensorRT) expect channels first: (3, H, W).

3

Channel duplication: 3 channels to 6

SUTrack was designed for multi-modal input (e.g., RGB + Depth, or RGB + Thermal). Each modality uses 3 channels. For pure RGB tracking, we simply duplicate the 3 RGB channels to produce a 6-channel input. This keeps the model architecture unchanged while using only a single camera.

Critical Rule 5 — RGB Input Only
The model was trained on RGB images. Always convert BGR (from OpenCV/GStreamer) to RGB with cv.cvtColor(frame, cv.COLOR_BGR2RGB) before calling preprocess(). A BGR input shifts every pixel's colors — the tracker silently produces wrong predictions.

Crop Extraction — sample_target()

Rather than processing the full frame, SUTrack only looks at a square crop centered on the target. This is more efficient and forces the model to focus on the relevant region. 

def sample_target(im, target_bb, search_area_factor, output_sz):
    x, y, w, h = target_bb
    crop_sz = math.ceil(math.sqrt(w * h) * search_area_factor)

    cx = x + 0.5 * w   # center of target
    cy = y + 0.5 * h

    x1 = round(cx - crop_sz * 0.5)   # crop top-left
    y1 = round(cy - crop_sz * 0.5)

    # Pad if the crop extends outside the image boundaries
    crop = im[max(y1,0):y1+crop_sz, max(x1,0):x1+crop_sz]
    crop = cv.copyMakeBorder(crop, y1_pad, y2_pad, x1_pad, x2_pad,
                              cv.BORDER_CONSTANT)
    crop = cv.resize(crop, (output_sz, output_sz))

    resize_factor = output_sz / crop_sz
    return crop, resize_factor

The crop size is determined by the target's size and a search area factor:

UseFactorCrop size formulaOutput size
Template (init)2.0sqrt(w*h) * 2.0112 x 112 px
Search region (each frame)4.0sqrt(w*h) * 4.0224 x 224 px

The resize factor returned is critical: it maps predicted coordinates from the 224-pixel crop space back to the original image space later in post-processing.

Border Padding
If the crop extends beyond the image edges (object near a border), the missing area is filled with black pixels using cv.copyMakeBorder. This prevents the crop from being smaller than expected, which would break the coordinate mapping.

Template Annotation — transform_image_to_crop()

When initializing, we also need to tell the model exactly where the target is inside the template crop. This is the template_anno input — a normalized [x1, y1, w, h] box in the 112x112 crop coordinate space. 

def transform_image_to_crop(box_in, box_extract, resize_factor, crop_sz):
    # Find center of target relative to crop center
    out_cx = (crop_sz - 1) / 2.0 + (cx_in - cx_ex) * resize_factor
    out_cy = (crop_sz - 1) / 2.0 + (cy_in - cy_ex) * resize_factor
    out_w  = w_in * resize_factor
    out_h  = h_in * resize_factor

    # Normalize to [0, 1] range
    norm = float(crop_sz - 1)
    return np.array([out_x1/norm, out_y1/norm, out_w/norm, out_h/norm])

The Hanning Window

The raw score map from the model can have secondary peaks — locations that look slightly like the target but are actually distractors (similar-looking objects in the background).

The Hanning (cosine) window is a 2D weight mask that is highest at the center and smoothly tapers to zero at the edges. Multiplying the score map by this window penalizes predictions far from the center of the search crop, suppressing edge distractors while preserving the true target peak. 

def hann1d(sz: int) -> np.ndarray:
    n = np.arange(1, sz + 1, dtype=np.float32)
    return 0.5 * (1.0 - np.cos(2.0 * math.pi * n / (sz + 1)))

def hann2d(h: int, w: int) -> np.ndarray:
    # Outer product of two 1D windows => 2D separable window
    return hann1d(h)[:, np.newaxis] * hann1d(w)[np.newaxis, :]

For the sutrack_t224 model the feature map is 14x14, so the window is a 14x14 array where the center cell has weight ≈1.0 and the corners have weight ≈0.0.

Critical Rule 4 — Hanning Window Before argmax
Multiply score_map * hann_window before finding the peak with np.argmax(). If you apply the window after argmax, it has no effect on which peak was selected — the suppression is completely bypassed.

TrackerInstance — Per-Object State

Each tracked object has its own TrackerInstance. It stores the template that was captured at initialization time and the current bounding box state. 

class TrackerInstance:
    object_id:       int           # unique integer ID
    state:           list          # [x, y, w, h] in image pixels
    _template_img:   np.ndarray    # (1, 6, 112, 112) float32 — frozen at init
    _template_anno:  np.ndarray    # (1, 4) float32 — normalized bbox in template
    last_seen_frame: int           # frame index of last successful update
    confidence:      float         # score_map peak value at last frame
    hann_window:     np.ndarray    # (14, 14) precomputed weight mask

initialize(frame_rgb, bbox, frame_idx)

Called once when the user picks a target. Extracts the template crop, preprocesses it, and computes the template annotation. These are stored and reused for every subsequent frame. 

def initialize(self, frame_rgb, bbox, frame_idx=0):
    self.state = list(bbox)

    # Extract and preprocess template crop
    z_crop, rf = sample_target(frame_rgb, self.state,
                               self.template_factor, self.template_size)
    self._template_img = preprocess(z_crop)   # (1, 6, 112, 112)

    # Compute normalized annotation: where is the target inside the template?
    anno = transform_image_to_crop(self.state, self.state, rf, self.template_size)
    self._template_anno = anno[np.newaxis].astype(np.float32)  # (1, 4)

Post-Processing — track()

This is where model outputs become a real bounding box. The method runs every frame. It is the most complex part of the tracking pipeline, so let's go step by step. 

def track(self, frame_rgb, engine, frame_idx=0):
    # 1. Extract search crop centered on last known position
    x_crop, resize_factor = sample_target(
        frame_rgb, self.state, self.search_factor, self.search_size)
    search_np = preprocess(x_crop)   # (1, 6, 224, 224)

    # 2. TRT inference
    trt_out = engine.infer(self._template_img, search_np, self._template_anno)

    # 3. Extract maps (already sigmoid-activated — do NOT sigmoid again)
    score_map  = trt_out['score_map'].reshape(self.feat_sz, self.feat_sz)
    size_map   = trt_out['size_map'].reshape(2, self.feat_sz * self.feat_sz)
    offset_map = trt_out['offset_map'].reshape(2, self.feat_sz * self.feat_sz)

    # 4. Apply Hanning window BEFORE argmax
    if self.hann_window is not None:
        score_map = score_map * self.hann_window

    # 5. Find peak location in the 14x14 grid
    idx   = int(np.argmax(score_map.reshape(-1)))
    idx_y = idx // self.feat_sz
    idx_x = idx % self.feat_sz

    # 6. Decode width, height, and sub-pixel offset at the peak
    w_norm = float(size_map[0, idx])
    h_norm = float(size_map[1, idx])
    off_x  = float(offset_map[0, idx])
    off_y  = float(offset_map[1, idx])

    # 7. Compute normalized center: grid position + sub-pixel offset
    cx_norm = (idx_x + off_x) / self.feat_sz
    cy_norm = (idx_y + off_y) / self.feat_sz

    # 8. Scale from normalized [0,1] -> crop-space pixels
    scale = self.search_size / resize_factor
    cx = cx_norm * scale
    cy = cy_norm * scale
    w  = w_norm  * scale
    h  = h_norm  * scale

    # 9. Map crop-space -> original image coordinates
    cx_prev = self.state[0] + 0.5 * self.state[2]
    cy_prev = self.state[1] + 0.5 * self.state[3]
    cx_real = cx + (cx_prev - 0.5 * scale)
    cy_real = cy + (cy_prev - 0.5 * scale)

    # 10. Position-only clamp: keep predicted w, h as-is
    x1 = max(0.0, min(cx_real - 0.5 * w, W - 1.0))
    y1 = max(0.0, min(cy_real - 0.5 * h, H - 1.0))

    self.state = [x1, y1, w, h]
    return self.state

Steps 6–9: Coordinate Decoding Explained

The model outputs everything in normalized coordinates relative to the feature map. Here is how we convert them to real pixel coordinates:

COORDINATE MAPPING 14x14 feature grid (normalized [0,1]) | | peak at grid cell (idx_x=7, idx_y=5) with offsets (0.3, -0.1) | v cx_norm = (7 + 0.3) / 14 = 0.521 cy_norm = (5 - 0.1) / 14 = 0.350 scale = search_size / resize_factor = 224 / (224 / crop_sz) = crop_sz (crop size in original image pixels) cx_crop = cx_norm * scale (center in crop-space pixels) cy_crop = cy_norm * scale cx_image = cx_crop + (cx_prev - 0.5 * scale) = cx_crop + (center_of_prev_bbox - half_of_crop_size) ^-- shift from crop-local to image-global coordinates
Critical Rule 3 — Position-Only Clamp
When the predicted box extends past the image boundary, clamp only the top-left corner (x1, y1). Never re-derive w and h from clamped corners.

Wrong approach: clamp x2 = min(x1+w, W), then set w = x2 - x1. This causes the box to shrink to a thin line whenever the target is near the image edge.

Correct approach: x1 = clamp(cx - w/2), keep w unchanged. The box may partially extend off-screen — that is fine and expected.
